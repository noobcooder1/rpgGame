## 2. 서버 실행 방법

### ✅ 올바른 실행 방법

> [!IMPORTANT]
> **반드시 `server/` 디렉토리 안에서** `npm start`를 실행해야 합니다!
> `rpg/` 루트 디렉토리에서 실행하면 `package.json`을 찾을 수 없어 오류가 발생합니다.

**터미널을 열고 아래 명령어를 순서대로 입력하세요:**

```bash
# 1단계: server 디렉토리로 이동
cd /Users/a1/workSpace/html:css,javascript/rpg/server

# 2단계: 서버 실행
npm start
```

**성공 시 출력:**
```
🔄 데이터베이스 초기화 중...
✅ 데이터베이스 생성/확인 완료
✅ users 테이블 생성/확인 완료
✅ game_saves 테이블 생성/확인 완료
✅ 데이터베이스 초기화 완료!

============================================
⚔️  RPG Game Server is running!
📍 Local: http://localhost:3001
🎮 Game:  http://localhost:3001/rpgmain.html
============================================
```

## 4. 자주 발생하는 오류 및 해결법

### ❌ 오류 1: `ENOENT - package.json을 찾을 수 없음`

```
npm error code ENOENT
npm error enoent Could not read package.json
```

**원인:** `rpg/` 루트 디렉토리에서 `npm start`를 실행한 경우

**해결:**
```bash
cd /Users/a1/workSpace/html:css,javascript/rpg/server
npm start
```

---

### ❌ 오류 2: `ECONNREFUSED - 데이터베이스 연결 실패`

```
❌ 서버 시작 실패: Error: connect ECONNREFUSED
```

**원인:** MySQL/MariaDB가 실행되지 않은 경우

**해결:**
```bash
# MariaDB 시작 (Homebrew 설치 시)
brew services start mariadb

# 또는 MySQL 시작
mysql.server start
```

---

### ❌ 오류 3: `모듈을 찾을 수 없음`

```
Error: Cannot find module 'express'
```

**원인:** npm 패키지가 설치되지 않은 경우

**해결:**
```bash
cd /Users/a1/workSpace/html:css,javascript/rpg/server
npm install
npm start
```

---

### ❌ 오류 4: `포트 이미 사용 중`

```
Error: listen EADDRINUSE :::3001
```

**원인:** 이미 3001 포트에서 다른 프로세스가 실행 중

**해결:**
```bash
# 3001 포트 사용 중인 프로세스 확인
lsof -i :3001

# 해당 프로세스 종료 (PID는 위 명령어에서 확인)
kill -9 <PID>

# 다시 서버 실행
npm start
```

## 5. 서버 구성 정보

| 항목 | 값 |
|------|-----|
| **서버 포트** | 3001 |
| **게임 주소** | http://localhost:3001/rpgmain.html |
| **API 주소** | http://localhost:3001/api |
| **DB 호스트** | localhost:3306 |
| **DB 이름** | rpg_game |
| **DB 계정** | root / root |

### 📁 서버 파일 구조

```
rpg/
├── server/                 ← 📌 이 폴더에서 npm start 실행!
│   ├── package.json        ← npm 설정 파일
│   ├── server.js           ← Express 서버 메인 파일
│   ├── db.js               ← MariaDB 연결 설정
│   ├── verify.js           ← 인증 미들웨어
│   ├── .env                ← 환경 변수 (DB 정보 등)
│   ├── routes/
│   │   ├── auth.js         ← 로그인/회원가입 API
│   │   └── save.js         ← 게임 저장/불러오기 API
│   └── node_modules/       ← npm 패키지 (자동 설치)
├── rpgmain.html            ← 게임 메인 페이지
├── auth.js                 ← 클라이언트 인증 시스템
└── ...
```

### 🔗 API 엔드포인트

| 기능 | 경로 | 메서드 |
|------|------|--------|
| 회원가입 | `/api/auth/register` | POST |
| 로그인 | `/api/auth/login` | POST |
| 로그인 확인 | `/api/auth/check` | GET |
| 로그아웃 | `/api/auth/logout` | POST |
| 게임 저장 | `/api/save` | POST |
| 게임 불러오기 | `/api/save` | GET |
